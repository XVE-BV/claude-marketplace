> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extend the existing `xve` plugin with a bash statusline that adds a handoff-urgency banner (amber at 60%, red at 85%, pace-bumped amber at 50%+⇡15%) on top of the claude-pace quota lines.

**Architecture:** Fork `claude-pace.sh` verbatim into `plugins/xve/statusline/xve-hud.sh`, preserve its two output lines, inject one new banner line above them driven by `PCT` (context %) and a newly computed 5h pace delta. Setup wired via extended `/xve-setup` plus a standalone `/xve-hud-setup`. Docs page added as a new `/hud` Vue route in `docs/index.html`.

**Tech Stack:** Bash + jq. No Node, no build step.

## Task 2: Inject the handoff-urgency banner

Compute the 5h pace delta inside the forked script (claude-pace computes it only inside its `_usage` helper — not exposed at the outer scope), then emit a banner line above the two existing lines whenever the urgency ladder triggers.

**Files:**
- Modify: `plugins/xve/statusline/xve-hud.sh`

### Banner logic

Inputs available at the end of the script (all already populated by the upstream jq call):
- `PCT` — context used percentage (integer, 0-100)
- `U5` — 5h used percentage (integer or `--`)
- `RM5` — remaining minutes in 5h window (integer, may be empty)

Compute pace delta for the 5h window (300 min total):

```
elapsed_5h      = 300 - RM5
expected_5h_pct = (elapsed_5h * 100) / 300
pace_delta_5h   = U5 - expected_5h_pct
```

If any of `U5`, `RM5` is non-numeric or empty, `pace_delta_5h` is treated as `0` (no bump).

Urgency ladder — evaluated top to bottom, first match wins:

| Condition                                                    | Banner                                               |
|--------------------------------------------------------------|------------------------------------------------------|
| `PCT >= 85`                                                  | `● handoff NOW — auto-compact imminent` (red)        |
| `PCT >= 60`                                                  | `● handoff soon — /session-handoff` (amber)          |
| `PCT >= 50` AND `pace_delta_5h >= 15`                        | `● handoff soon — /session-handoff` (amber)          |
| *(otherwise)*                                                | *(no banner line printed)*                           |

Immediately **before** the two `printf` lines at the bottom, insert the banner block. Find the literal text `printf '%s\n' "$L1"` and insert this above it (exact placement — use a larger surrounding match if needed for uniqueness):

```bash
# ── Handoff urgency banner (XVE) ──
# Thresholds mirror /help canon: <60 quiet, 60-85 amber, >=85 red.
# Pace bump: >=50% ctx AND >=+15% pace delta on the 5h window bumps to amber.
BANNER=""
if [[ "$PCT" =~ ^[0-9]+$ ]]; then
  _pace_delta_5h=0
  if [[ "$U5" =~ ^[0-9]+$ ]] && [[ "$RM5" =~ ^[0-9]+$ ]]; then
    _elapsed_5h=$((300 - RM5))
    if ((_elapsed_5h < 0)); then _elapsed_5h=0; fi
    _expected_5h=$((_elapsed_5h * 100 / 300))
    _pace_delta_5h=$((U5 - _expected_5h))
  fi
  if ((PCT >= 85)); then
    BANNER="${R}● handoff NOW — auto-compact imminent${N}"
  elif ((PCT >= 60)); then
    BANNER="${Y}● handoff soon — /session-handoff${N}"
  elif ((PCT >= 50)) && ((_pace_delta_5h >= 15)); then
    BANNER="${Y}● handoff soon — /session-handoff${N}"
  fi
fi
[ -n "$BANNER" ] && printf '%s\n' "$BANNER"
```

(`$R`, `$Y`, `$N` are the ANSI color vars already defined near line 24 of the script.)
