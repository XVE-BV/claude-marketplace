XVE Setup Guide

XVE · Run once on a new machine to wire up Claude Code with the full XVE configuration

Type /xve-setup in Claude Code after installing the plugin. It walks through each step, asks before touching anything, and shows a checklist at the end.

What Each Step Does

1
Find repo location Detects where the marketplace is checked out so it can reference files. Asks if it can't find it automatically.
2
Apply settings.json Merges XVE config into ~/.claude/settings.json. Sets model strategy (Sonnet executor + Opus advisor), permissions, and token limits.
3
Install hooks (asks first) session-start.sh — injects context at the start of every session.
env-guard.sh — blocks Claude from reading or writing .env files.
4
Check env vars Reports which vars are set and which are missing. Get values from Bitwarden: "xve claude marketplace env vars".
5
Install karpathy-skills Installs four coding principles automatically — no prompt needed.
6
Summary checklist Prints what was applied, what was skipped, and what still needs attention.

Environment Variables

Variable Purpose
XVE_EMAIL Your email — used by Claude to identify your account
DISABLE_ADVISOR Set to 1 to turn off Opus advisor calls
Missing a var? Add it to ~/.zshrc and restart your terminal. Get values from Bitwarden: "xve claude marketplace env vars".
